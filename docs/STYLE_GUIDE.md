# Style Guide: HoneyPot

## General Principles
- **Readability first**: Write code for other humans, not just the compiler.
- **Consistency**: Follow the project's established patterns.

## Language-Specific Guidelines (Dart)
_Adopt standard community practices for Dart._

- **Naming**: Use camelCase for variables, PascalCase for classes (adjust based on Dart norms).
- **Indentation**: Use 4 spaces (or 2 if standard for Flutter).
- **Comments**: Focus on 'Why', not 'What'.
