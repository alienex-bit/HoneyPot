# Contributing to HoneyPot

## Getting Started
1. Fork the repository and create a feature branch: `git checkout -b feature/your-feature`
2. Make your changes, following the coding conventions (see `STYLE_GUIDE.md`).
3. Write or update tests in `tests/` (see `TESTING.md`).
4. Open a pull request with a clear description of your changes.

## 📝 Coding Conventions
- Language: **Dart**
- Keep functions small and focused on one responsibility.
- Write self-documenting code; add comments only where intent is non-obvious.

## 💾 Commit Messages
Use the conventional commits format:
```text
feat: add login page
fix: correct off-by-one in pagination
docs: update README setup steps
```
