# Testing Strategy: HoneyPot

## Philosophy
We aim for high test coverage on core business logic. All pull requests must include relevant tests.

## Running Tests
Based on your stack (Flutter), use the following commands:

- **Unit Tests**: Run tests in the `tests/` directory.
- **Continuous Integration**: See `.github/workflows/` (if enabled).

## Test Organization
- `tests/unit/`: Logic and utility tests.
- `tests/integration/`: Service and interaction tests.
